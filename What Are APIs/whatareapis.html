What are APIs?

API stands for Application Programming Interface:

  • A - Application
  • P - Programming
  • I - Interface
    • Providing the developer a means of controlling their interactions while hiding away all the complex and abstract stuff. We don't need to know how an API works, and the interface hides all of the complex and abstrct code from us (developers) so we don't "reinvent the wheel".
    • As a developer, we need to know what the API can do for us and how we can integrate it into our project.
    • The interface defines ways for users to interact or communicate with an object (a GUI is a good example).

APIs are essentially a set of instructions on how to interact with specific sites/apps/services. APIs are like a contract where they define how they will be used and defines what is expected to be received when being used. APIs are mostly web-based, but there are ones that are local.

Frameworks are API(s) that provide major customization and scalability to make it your own

Remote APIs are very good for the purpose of removing the limitation of computational power of a smartphone (for example).

We use APIs every day to interact with a lot of different sources to obtain data. For example, the weather app on the iPhone uses an API to pull data from the Google weather services to leverage in their app. Another example of an API is the Spotify app on your phone using an API to pull the song you want to listen to from their servers and stream it to your device. A third, and probably very common, example is YouTube. They use APIs to load the videos in the web browser and stream the videos from their servers.

There are two types of major APIs: REST (often called RESTful) and SOAP. SOAP came before REST and was invented by Microsoft. There are benefits to both, but REST is used to develop over 70% of public APIs. Powerhouse companies like Google and Amazon use REST APIs in their technologies.
Here are the benefits to both SOAP and REST:

  • REST (Representational State Transfer)
    • REST allows for a variety of data formats while SOAP only allows XML.
    • REST uses lower bandwidth, making these APIs faster when connecticting to remote servers/services/etc.
    • REST is designed to access data while SOAP is designed to perform operations through a standardized set of messaging patters.
    • REST is considered easy to work with, and thanks to JSON, REST offers superior support for browser clients.
  • SOAP (Simple Object Access Protocol)
    • SOAP's standard HTTP protocol makes it easier for it to operate across firewalls and proxys.
    • SOAP supports WS-Security, which is invauluable when in need of data privacy and integrity.
    • SOAP has builti-in retry logic that prevents the user from manually retrying the API call the first time it fails.
    • SOAP has a standard set of rules, which means that the service and consumer don't necesarily have to have the same understanding of content and context.
